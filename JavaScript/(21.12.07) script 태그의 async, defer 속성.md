# script 태그의 async, defer 속성

## script 태그
- 사용 위치


- ```<script>```
- Async 속성 사용
- 기본값이 true라서, async만 입력하면 됨
- Head 태그에 사용
- HTML 파싱 하다가 script 코드 보고 병렬적으로 js. 파일 실행시킴
- Js 파일 다운로드 완료 되면 html 파싱 멈추고 js 파일 실행
- 실행되면 마저 HTML 파싱 진행

- 파싱하는 동안 멈출 수 있으며, js 파일에 필요한 엘리먼트가 아직 HTML에서 정의되지 않을 수 있음


- Deafer 속성 사용
- 병렬적으로 js 파일 다운로드
- HTML 파싱이 끝나면 이어서 JS 파일 바로 실행


Async vs defer
- async는 여러 개의 script 태그 사용 시, 코드 순서 상관없이 먼저 다운로드 된 Js 파일 실행
- defer은 HTML parsing 하는 동안, 여러개의 js 파일이 다운로드 되고, 이후 script 태그의 코드 순서대로 파일 실행



**
`use strict`  : 이것을 js 파일 위에 선언하면, ECMA 5에 사용됨
- JS 파일의 유연성 증대
- 정의되지 않은 변수를 사용할 수 없음
- Js 엔진이 효율적으로 분석하여 성능 개선
